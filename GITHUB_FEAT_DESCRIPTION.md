### 2. 🌏 港股完整支持与智能币种识别

#### 问题
原有系统仅支持美股(A股)，且币种识别规则过于简单(.SS/.SZ为人民币，其余为美元)。

#### 解决方案
- **全面支持港股(HK)**: 扩展行情数据源以覆盖香港交易所(.HK结尾股票)
- **三币种智能识别**: 完善[`detect_currency_from_symbol()`](backend/app/services/price_service.py:24-39)函数
  - `.SS` / `.SZ` → CNY (人民币)
  - `.HK` → HKD (港币)  
  - 其他符号 → USD (美元)
- **实时行情兼容**: 确保 Yahoo Finance API 对三种市场的数据获取稳定可靠

#### 使用示例
```yaml
watchlist:
  - symbol: "0700.HK"    # 腾讯控股 - 港币计价
    name: "腾讯控股"
    market: hk
  - symbol: "600000.SH"  # 浦发银行 - 人民币计价
    name: "浦发银行"
    market: cn
  - symbol: AAPL         # 苹果公司 - 美元计价
    name: "苹果公司"
    market: us
```

### 4. 👤 企业级用户认证系统

#### 必要性
生产环境下的投资管理系统需要严格的多租户隔离和安全管控。

#### 功能清单
##### 4.1 用户注册与管理
- **POST `/api/auth/register`**: 新用户注册
  - 用户名唯一性校验
  - 邮箱格式验证与去重
  - BCrypt密码加密存储
  
##### 4.2 身份认证与会话管理
- **POST `/api/auth/login`**: 登录鉴权
  - JWT令牌颁发机制
  - 24小时有效期配置
  - 账号状态激活检查
  
##### 4.3 个人中心操作
- **GET `/api/auth/me`**: 获取当前用户资料
- **POST `/api/auth/change-password`**: 修改密码
  - 旧密码二次确认
  - 新旧密码差异校验
- **POST `/api/auth/logout`**: 会话注销

#### 技术亮点
- 采用OAuth2 Password Flow标准协议
- HS256算法JWT签名
- bcrypt自适应哈希强度
- 时区感知的时间戳处理

#### 相关模型扩展
在 [`models.py`](backend/app/models.py) 中新增Pydantic Schema:
- `UserRegister`: 注册请求DTO
- `UserLogin`: 登录请求DTO  
- `ChangePassword`: 密码修改DTO
- `UserInfo`: 用户信息响应DTO

# 3. 受保护资源访问

  -H "Authorization: Bearer $TOKEN"
