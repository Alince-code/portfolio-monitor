# Portfolio Monitor 运行文档

## 项目简介

Portfolio Monitor 是一个股票监控工作台，支持美股和 A 股实时行情，多档位目标价告警（Telegram 推送），持仓和交易记录管理等功能。

## 快速运行

### 方法一：使用本地开发模式

1. **复制配置文件**
   ```bash
   cp config.example.yaml config.yaml
   ```

2. **安装后端依赖**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r backend/requirements.txt
   ```

3. **安装前端依赖并构建**
   ```bash
   cd frontend
   npm install
   npm run build
   cd ..
   ```

4. **启动服务**
   - 前台运行（开发调试）：`bash start.sh`
   - 后台运行：`bash run.sh`
   - 停止后台进程：`bash stop.sh`

5. **访问项目**
   打开浏览器访问：http://localhost:8802

### 方法二：使用 Docker 部署

1. **复制配置文件**
   ```bash
   cp config.example.yaml config.yaml
   ```

2. **编辑配置文件**
   填入你的 Telegram Bot Token 和 Chat ID（可选）

3. **启动容器**
   ```bash
   docker compose up -d
   ```

4. **访问项目**
   打开浏览器访问：http://localhost:8802

5. **常用 Docker 命令**
   - 查看日志：`docker compose logs -f`
   - 重启服务：`docker compose restart`
   - 停止服务：`docker compose down`
   - 重新构建并启动：`docker compose up -d --build`

## 前端开发模式

如果需要进行前端开发，可以使用 Vite 的热更新模式：

```bash
cd frontend
npm run dev
```

此时前端服务运行在 http://localhost:5173，会自动代理 /api 请求到后端。

## 配置说明

配置文件为 `config.yaml`，主要配置项如下：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `server.host` | 监听地址 | `0.0.0.0` |
| `server.port` | 监听端口 | `8802` |
| `database.url` | SQLite 数据库路径 | `sqlite:///data/portfolio.db` |
| `monitor.interval_active` | 盘中刷新间隔（秒） | `30` |
| `monitor.interval_idle` | 盘外刷新间隔（秒） | `300` |
| `monitor.change_alert_pct` | 涨跌幅告警阈值（%） | `5.0` |
| `telegram.bot_token` | Telegram Bot Token（从 @BotFather 获取） | — |
| `telegram.chat_id` | 推送目标（群组 ID 为负数，个人 ID 为正数） | — |
| `telegram.thread_id` | Forum 群组话题 ID（非 Forum 群可删除） | — |
| `alert.cooldown` | 同一告警冷却时间（秒） | `3600` |
| `watchlist` | 默认监控标的列表（也可通过 Web 界面管理） | — |

## API 文档

后端基于 FastAPI，提供了交互式 API 文档：

- **Swagger UI**：http://localhost:8802/docs
- **ReDoc**：http://localhost:8802/redoc

## 技术栈

- **后端**：Python 3.11 + FastAPI + SQLAlchemy + APScheduler
- **前端**：Vue 3 + Vite + ECharts 5
- **数据库**：SQLite
- **行情数据**：Yahoo Finance（美股）/ AKShare（A股）
- **告警推送**：Telegram Bot API

## 功能特性

- **实时行情**：美股 + A 股，自动刷新（盘中 30s / 盘外 5min）
- **多档位告警**：每只股票支持主档 + 多个子档（买入 / 卖出 / 止损），Telegram 即时推送
- **持仓管理**：自动计算成本、盈亏、收益率，按市场分组，ECharts 图表可视化
- **交易记录**：增删查改、筛选、CSV 导出
- **现金账户**：USD / CNY 现金管理，交易自动联动
- **双主题**：暗色 / 明色主题，跟随系统设置，一键切换
- **响应式**：桌面端 & 移动端完美适配

## 故障排查

1. **端口占用**：如果端口 8802 被占用，可以修改 `config.yaml` 中的 `server.port` 配置。

2. **依赖安装失败**：
   - 确保 Python 版本为 3.9+ 
   - 确保 npm 版本为 6+ 
   - 可以尝试使用国内镜像源加速依赖安装

3. **行情数据获取失败**：
   - 检查网络连接
   - 对于 A 股数据，可能需要国内网络环境

4. **Telegram 推送失败**：
   - 确保 Bot Token 正确
   - 确保 Chat ID 正确（群组 ID 为负数）
   - 确保 Bot 已经加入群组并获得发送消息的权限

## 注意事项

1. **数据安全**：本项目使用 SQLite 数据库，数据存储在本地 `data/portfolio.db` 文件中，请定期备份。

2. **性能优化**：
   - 监控的股票数量不宜过多，建议不超过 50 只
   - 盘中刷新间隔不宜设置过短，建议不低于 10 秒

3. **网络使用**：
   - 实时行情会定期请求外部 API，可能产生一定的网络流量
   - 建议在稳定的网络环境下使用
